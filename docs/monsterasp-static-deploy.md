# تعليمات رفع المشروع على MonsterASP (Static Export)

هذا الدليل يشرح كيفية رفع مشروع Kids Arabic Educational Games على MonsterASP باستخدام static export.

## المتطلبات

- حساب على MonsterASP
- وصول FTP أو File Manager
- Node.js مثبت محليًا للبناء

## الخطوات

### 1. بناء المشروع محليًا

```bash
# تثبيت التبعيات
npm install

# بناء المشروع للإنتاج
npm run build
```

بعد اكتمال البناء، تأكد من وجود مجلد `out/` في جذر المشروع.

### 2. رفع الملفات على MonsterASP

#### الملفات المطلوبة رفعها:

ارفع محتويات مجلد `out/` بالكامل إلى `/wwwroot` على MonsterASP.

**لا ترفع:**
- `node_modules/`
- `.next/`
- `.git/`
- `dev-server.log`
- `dev-server.err.log`

#### ملف web.config

انسخ الملف:
```
deployment/monsterasp-static/web.config
```

إلى:
```
/wwwroot/web.config
```

هذا الملف ضروري لـ:
- إعدادات MIME types للملفات الصوتية (mp3, ogg)
- إعدادات MIME types للصور (webp, svg)
- إعدادات rewrite rules للـ SPA routing
- دعم JSON و webmanifest

### 3. التحقق من الرفع

بعد الرفع:
1. افتح الموقع في المتصفح
2. تأكد من تحميل الصفحة الرئيسية
3. تأكد من عمل التنقل بين الدروس
4. تأكد من تشغيل الصوتيات

### 4. استكشاف الأخطاء

#### إذا لم تعمل الصفحة:
- تأكد من وجود `web.config` في `/wwwroot`
- تأكد من رفع كل محتويات `out/`
- تحقق من سجلات الأخطاء في MonsterASP

#### إذا لم تعمل الصوتيات:
- تأكد من إعدادات MIME types في `web.config`
- تأكد من وجود ملفات الصوت في `public/audio/`
- تحقق من مسارات الصوت في المتصفح

#### إذا لم تعمل الصور:
- تأكد من رفع مجلد `public/images/`
- تأكد من إعدادات MIME types في `web.config`

## ملاحظات مهمة

- **لا تشغل `npm start`** على السيرفر - هذا static export
- **لا ترفع `node_modules`** - غير مطلوبة
- **لا ترفع `.next`** - تم تحويله إلى `out/`
- المشروع يعمل بالكامل على جانب العميل
- البيانات محفوظة في localStorage في المتصفح

## المزايا

- سريع جداً (ملفات static)
- لا يحتاج Node.js على السيرفر
- استهلاك منخفض للموارد
- سهل الصيانة

## العيوب

- لا يمكن تحديث البيانات بدون إعادة البناء
- لا يمكن إضافة دروس جديدة بدون إعادة البناء
- localStorage محدود بالحجم والمتصفح

## الدعم

إذا واجهت مشاكل:
1. تحقق من سجلات IIS في MonsterASP
2. تأكد من إعدادات MIME types
3. تأكد من صحة مسارات الملفات
