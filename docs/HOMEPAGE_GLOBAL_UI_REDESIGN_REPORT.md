# تقرير إعادة تصميم الصفحة الرئيسية بواجهة جامعية عالمية

## 1. الملخص التنفيذي

تم تنفيذ إعادة تصميم كاملة لواجهة الصفحة الرئيسية (`/`) لتظهر كواجهة **جامعة إلكترونية رسمية** ذات طابع مؤسسي أكاديمي، مع الحفاظ على هوية الموقع كموقع تعريفي للجامعة وليس LMS.

النتيجة الحالية:
- مظهر أكثر جدية واحترافية.
- إبراز واضح لهوية الجامعة الإلكترونية.
- مسار قبول وبرامج أكاديمية أوضح.
- تقليل الإحساس بالبوابة/المنتج وتحويله إلى واجهة جامعة عالمية الطابع.

## 2. سبب تغيير الصفحة الرئيسية

النسخة السابقة كانت أقرب بصريًا إلى طابع "بوابة رقمية/منتج" أكثر من كونها واجهة جامعة مؤسسية عالمية. الهدف من التعديل كان:

1. تعزيز الثقة المؤسسية.
2. إبراز هوية الجامعة الإلكترونية رسميًا.
3. تنظيم المسارات الأكاديمية والقبول بشكل واضح.
4. إبقاء الأخبار عنصرًا داعمًا وليس المسيطر.
5. تقديم واجهة مناسبة للعرض على الإدارة والعملاء.

## 3. ما تم تغييره في UI

1. إعادة بناء الهيكل البصري للصفحة من الصفر داخل `Home.tsx`.
2. إعادة ضبط الـHero ليعبر عن جامعة إلكترونية رسمية (Institutional + Online identity).
3. اعتماد بطاقات مؤسسية متناسقة وخفيفة بصريًا بدل طابع product/portal.
4. تقليل الزخرفة البصرية والانيميشن المبالغ فيه.
5. تحسين التسلسل البصري للعناوين والأقسام وCTAs.
6. الحفاظ على توافق RTL/LTR عبر `useLanguage`.

## 4. الهيكل الجديد للصفحة الرئيسية

تم تطبيق الهيكل المطلوب بالترتيب:

1. Institutional Online University Hero
2. Quick Access
3. Electronic University Model
4. Academic Study Pathways
5. Faculties and Programs
6. Trust and Institutional Credibility
7. Research, Journals, and Centers
8. Admissions Journey
9. News and Events
10. Final CTA

## 5. العناصر التي تم حذفها أو تقليلها

1. تقليل الطابع الذي يوحي بواجهة منتج/بوابة تقنية.
2. تقليل الكتل التي تركز على "إحصائيات سريعة" كعنصر أساسي للانطباع الأول.
3. تقليل الإيحاءات اللغوية التي تقترب من أسلوب LMS أو marketplace.
4. تقليل التأثيرات البصرية الثقيلة لصالح أسلوب مؤسسي هادئ.

## 6. العناصر التي تم إضافتها

1. نبرة Hero جديدة توضح هوية الجامعة الإلكترونية بشكل مباشر.
2. قسم جديد: **نموذج الجامعة الإلكترونية** يشرح:
   - مرونة الدراسة عبر الإنترنت
   - الوصول من أي مكان
   - تنظيم البرامج حسب المراحل
   - التواصل الأكاديمي المؤسسي
   - مسار قبول واضح
   - الشهادات والاعتمادات
3. تحسين أقسام الاكتشاف الأكاديمي (الكليات + البرامج) بتجربة أكثر مؤسسية.
4. تحسين CTA النهائي ليدفع نحو البرامج والقبول والتواصل بشكل واضح.

## 7. توافق التصميم مع مواقع الجامعات العالمية

التصميم الآن أقرب لمبادئ واجهات الجامعات العالمية من حيث:

1. Hero مؤسسي قوي يوضح هوية الجامعة مباشرة.
2. تنظيم أكاديمي واضح (Programs/Admissions/Faculties).
3. وجود قسم ثقة مؤسسي رسمي.
4. إبراز البحث والمجلات والمراكز كجزء من هوية الجامعة.
5. تقليل الضوضاء البصرية وزيادة وضوح المسارات.

## 8. دعم العربية والإنجليزية وRTL/LTR

1. جميع النصوص في الصفحة الرئيسية ثنائية اللغة عبر `t(ar, en)`.
2. اتجاه الواجهة (RTL/LTR) محفوظ ويعمل حسب اللغة الحالية.
3. اتجاه الأسهم وروابط التنقل متوافق مع RTL/LTR.
4. لم يتم كسر سلوك تبديل اللغة الحالي.

## 9. تأثير التغيير على SEO

1. تم تحسين وصف الصفحة الرئيسية في `Seo.tsx` ليعكس:
   - Official university website
   - Academic programs
   - Admissions
   - Faculties
   - Research
   - Institutional information
2. لم يتم إدخال claims غير قابلة للتحقق.
3. تم الحفاظ على نظام SEO الحالي دون كسر بنية الـcanonical/og.

## 10. أوامر التحقق ونتائجها

| الأمر | النتيجة | ملاحظات |
|---|---|---|
| `pnpm run typecheck` | نجح | لا توجد أخطاء TypeScript |
| `pnpm --filter @workspace/university run build` | نجح | تحذيرات sourcemap قديمة + تحذير حجم chunk غير مانع |
| `pnpm --filter @workspace/api-server run build` | نجح | لم يتأثر الباكند بالتعديلات |
| `pnpm run lint` | غير متاح | لا يوجد script مخصص حتى الآن |
| `pnpm run test` | غير متاح | لا يوجد script مخصص حتى الآن |

## 11. ملاحظات متبقية إن وجدت

1. لم أتمكن من تشغيل معاينة محلية (`vite dev/preview`) داخل البيئة الحالية بسبب خطأ بيئي `spawn EPERM`، لذلك تم الاعتماد على build/typecheck للتحقق التقني.
2. يمكن لاحقًا تحسين الصور الرسمية للـHero والأقسام عند توفر أصول بصرية معتمدة من الجامعة.
3. لم يتم تعديل أي جزء LMS أو إضافة أي وظيفة خارجة عن نطاق الموقع المؤسسي.
