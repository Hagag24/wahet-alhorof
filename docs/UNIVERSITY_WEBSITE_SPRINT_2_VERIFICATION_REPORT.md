## 2. التحقق من مسارات API المؤسسية

### 2.1 فصل API العامة والمحمية

تم التحقق من الملف:
- `artifacts/api-server/src/routes/institutional.ts`

**النتيجة:** ✅ تم التحقق بنجاح

**التفاصيل:**

**المسارات العامة (بدون مصادقة):**
- `GET /institutional/president-message` - يعرض رسالة منشورة واحدة فقط
- `GET /institutional/council` - يعرض نظرة عامة وأعضاء المجلس النشطين
- `GET /institutional/trust-items?type=...` - يعرض عناصر الثقة المنشورة (مع فلترة حسب النوع)
- `GET /institutional/certificates` - يعرض الشهادات المنشورة
- `GET /institutional/centers` - يعرض المراكز المنشورة
- `GET /institutional/centers/:slug` - يعرض تفاصيل مركز محدد
- `GET /institutional/journals` - يعرض المجلات المنشورة
- `GET /institutional/footer-links` - يعرض روابط الفوتر المنشورة

**جميع المسارات العامة:**
- تُرجع فقط المحتوى بـ `status = 'published'` (أو `active` للأعضاء)
- تُرجع `null` أو مصفوفة فارغة إذا لم يكن هناك محتوى
- لا تعرض أخطاء تقنية للمستخدم العام

**المسارات المحمية (تتطلب مصادقة وأدوار):**
- جميع مسارات `/admin/institutional/*` تستخدم `requireAuth` و `requireRole`
- الأدوار المسموح بها: `super_admin`, `admin`, `content_manager`, `editor`
- عمليات الحذف تتطلب `super_admin` أو `admin` فقط
- تدعم CRUD كامل (Create, Read, Update, Delete) لكل وحدة

**التحقق من المدخلات:**
- دوال `payloadBuilder` لكل وحدة تتحقق من:
  - الحقول المطلوبة
  - أنواع البيانات الصحيحة
  - القيم المسموحة للحالة والنوع
  - التحقق من slug للمراكز والمجلات
- إرجاع خطأ 400 إذا كانت البيانات غير صالحة

**لا توجد مشاكل:** الفصل بين API العامة والمحمية صحيح وآمن.

- `/centers/:slug` - تفاصيل مركز محدد
- `/journals` - المجلات العلمية

**السلوك الاحتياطي (Fallback):**
- جميع الصفحات تحتوي على محتوى احتياطي آمن بالعربية والإنجليزية
- رسالة توضيحية عند استخدام المحتوى الاحتياطي: "سيتم تحديث البيانات الرسمية بعد اعتمادها من إدارة الجامعة."

**دعم RTL/LTR:**
- جميع المكونات تستخدم `useLanguage` للتبديل بين العربية والإنجليزية
- `dir={lang === "ar" ? "rtl" : "ltr"}` في التخطيط الرئيسي
- محاذاة النصوص تتغير حسب اللغة

**صفحة تفاصيل المركز:**
- إذا لم يتم العثور على المركز (404)، تُستخدم `ContentPage` كاحتياطي
- هذا يسمح بإنشاء صفحات مخصصة عبر CMS للمراكز

**ISSN في المجلات:**
- حقل ISSN اختياري في المخطط
- يُعرض فقط إذا كان موجودًا في البيانات
- التنسيق: `ISSN: {value}`

**لا توجد مشاكل:** الصفحات العامة تستخدم بيانات CMS بشكل صحيح مع احتياطات آمنة ودعم RTL/LTR كامل.
